# Contributing

## Setup

```bash
git clone https://github.com/Ormorshtein/applicative-load-observability.git
cd applicative-load-observability
uv pip install -e ".[dev]"
pre-commit install
```

## Branching

Branch from `develop`, not `main`. Use descriptive names:

```
feature/scroll-detection
fix/bulk-took-nanoseconds
refactor/shared-latency-tracker
```

Merge to `develop` first. `main` tracks releases.

## Code

- Python 3.11+, enforced by ruff and mypy
- Modules stay under ~300 lines. Split by responsibility when they grow.
- Entry points get clean names (`setup.py`, `stress.py`). Internal modules get `_` prefix (`_client.py`, `_engine.py`).
- Tests go in `tests/unit/<component>/` mirroring the source tree.

### Lint, format, type-check

```bash
ruff check .
ruff format .
mypy analyzer/
```

### Run tests

```bash
pytest                    # all unit tests
pytest tests/unit/analyzer/  # one component
pytest -n auto            # parallel
```

## Docker images

There are **6 images**. Build and push **all of them** on every release, even if only one changed. Tags follow `<component>-<version>`.

```bash
# Build
docker build -t oracle1012/applicative-load-observability:analyzer-1.18.1 -f analyzer/Dockerfile .
docker build -t oracle1012/applicative-load-observability:logstash-1.18.1 -f logstash/Dockerfile .
docker build -t oracle1012/applicative-load-observability:gateway-1.18.1 -f gateway/Dockerfile gateway/
docker build -t oracle1012/applicative-load-observability:kibana-setup-1.18.1 -f kibana/Dockerfile kibana/
docker build -t oracle1012/applicative-load-observability:grafana-setup-1.18.1 -f grafana/Dockerfile grafana/
docker build -t oracle1012/applicative-load-observability:stress-1.18.1 -f tools/stress/Dockerfile .

# Push
for img in analyzer logstash gateway kibana-setup grafana-setup stress; do
  docker push oracle1012/applicative-load-observability:${img}-1.18.1
done
```

Note the build contexts: gateway and kibana/grafana-setup build from their own directories, not the repo root.

## Releasing

1. Update `version` in `pyproject.toml`
2. Update image tags in `docker-compose.yml` (3 services: analyzer, logstash, gateway)
3. Add a section to `CHANGELOG.md` with subsections: Analyzer, Dashboard, Infrastructure, Documentation
4. Build and push all 6 Docker images
5. Commit, push to `develop`, merge to `main`

## Dashboards

### Grafana

Dashboards are JSON files under `grafana/provisioning/dashboards/`. They're generated by Python builders:

```bash
# Regenerate from builders
cd grafana && python -c "from _dashboards import export_dashboards; export_dashboards()"
```

Restart Grafana to pick up changes (`docker compose restart grafana`).

### Kibana

Dashboards are Lens saved objects created via the Kibana API:

```bash
# Rebuild all dashboards + export ndjson
python kibana/setup.py --rebuild --elasticsearch http://127.0.0.1:9201
```

This creates data views, upserts visualizations, builds dashboards, and exports `.ndjson` files. The ndjson files are committed so `setup.py` (without `--rebuild`) can import them directly.

### Parity

Both Grafana and Kibana dashboards should show equivalent panels. When adding panels, update both:
- `grafana/_dashboard_builders.py` (Grafana)
- `kibana/_dashboard_builders.py` (Kibana)

## Helm chart

The Helm chart lives in `helm/alo/`. Any change to Docker Compose config, Logstash pipeline, or environment variables **must** also update the corresponding Helm templates.

Key files:
- `helm/alo/templates/logstash/configmap.yaml` — mirrors `logstash/pipeline/observability.conf` and `logstash/logstash.yml`
- `helm/alo/templates/gateway/deployment.yaml` — mirrors gateway env vars
- `helm/alo/values.yaml` — default values
- `helm/alo/values.schema.json` — must be updated when adding new values

Verify with `helm template test helm/alo/` before committing.

## Project structure

```
analyzer/           FastAPI service (stress scoring, record building)
gateway/            OpenResty reverse proxy (Lua, fire-and-forget to Logstash)
logstash/           Pipeline config (receives from gateway, calls analyzer, writes to ES)
kibana/             Dashboard setup scripts + Lens builders
grafana/            Dashboard builders + provisioning JSON
shared/             Shared Python utilities (HTTP client, data generators)
tools/stress/       Load generation tool (11 workload profiles)
tests/unit/         Unit tests (analyzer/, challenges/, shared/)
tests/integration/  Gateway resilience tests
tests/challenges/   Interactive challenge scenarios (14 total)
helm/alo/           Kubernetes Helm chart
docs/               Architecture, Helm guide, product spec
```
